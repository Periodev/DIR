# DIR (AOW) — Game Rules Specification

> 供 Claude Code 實作參考。所有設計決策均已確認，標注「待定」者除外。

---

## 0. 一句話定義

DIR 是一個節點路徑謎題：玩家在地圖上移動，取得帶方向的向量並攜帶，將其部署到正確位置以接通路線，最終抵達終點。

核心循環：**取得路線 → 移動到接點 → 接上路線**

---

## 1. 地圖結構

### 1.1 元素

| 元素 | 說明 |
|------|------|
| **節點（Node）** | 地圖上的可站立點，玩家只能站在節點上 |
| **邊（Edge）** | 連接兩個節點的線段，代表可能的移動路徑 |
| **起點 S** | 玩家初始位置 |
| **終點 G** | 抵達即過關 |

### 1.2 地圖標記符號（設計用）

- `-`、`x`、`|`：連通符號，表示邊存在，**不是節點**
- 字母 / 數字：節點
- 範例：`S - 1 - G` 表示三個節點由兩條邊連接

---

## 2. 移動規則

- 玩家可沿**已存在且可通行的邊**在節點間移動
- **移動本身不需要消耗向量**
- 移動不產生任何資源，也不觸發登錄
- 單向邊：只能從規定方向移動（見第 5 節）

---

## 3. 向量（Vector）

### 3.1 本質

向量是**帶方向的能力**，代表「從某個起點節點出發，向某方向行進的能力」。

- 向量有固定方向（上 / 下 / 左 / 右）
- **向量不可旋轉**，取得時方向即固定
- 向量與取得位置脫鉤——從 A 點取得的向右向量，可在地圖上任何合法位置釋放

### 3.2 取得（登錄）

| 規則 | 說明 |
|------|------|
| **必須移動到節點** | 只有站在節點上，才能登錄該節點的方向向量 |
| **不需要走進邊** | 站在節點、觀察到邊的存在即可登錄，不需要實際走進去 |
| **端點對應方向** | 登錄哪個方向，取決於玩家站在邊的哪一端：<br>• 要取「向右」→ 必須站在該邊的**左端點**<br>• 要取「向下」→ 必須站在該邊的**上端點**<br>• 以此類推（站在出發側） |
| **只能登錄實際存在的邊** | 節點沒有某方向的邊，則無法登錄該方向 |

### 3.3 使用（釋放）

- 站在某邊的對應起點，釋放方向匹配的向量 → 接通該邊
- 向量**使用後消耗**，不可重複使用
- 釋放成功後，該邊從不通變為可通行

### 3.4 未接通的邊

- 地圖上部分邊預設不可通行（需要向量才能激活）
- 部分邊預設可通行（玩家可自由移動）
- 兩種邊的視覺區分需在實作時定義（建議：亮度 / 顏色）

---

## 4. 庫存（Inventory）

- 玩家同時最多持有 **2 個向量**
- 要登錄第 3 個，必須先釋放或放棄其中一個
- 設計意圖：2 格剛好對應「一個開路、一個保留回程」的最小對稱結構

---

## 5. 單向限制

### 5.1 統一規則

移動和登錄共用**同一條單向限制**：

- 邊標記為單向時，**只能從規定方向移動，也只能從規定端點登錄**
- 不存在「可以走但不能登錄」或「可以登錄但不能走」的分裂狀態

### 5.2 視覺表示（參考 Linelight）

| 視覺元素 | 語意 |
|----------|------|
| 亮度 | 邊是否可通行 |
| 線段上的箭頭 | 單向方向 |
| 走過後變色 | 已使用 / 已接通的視覺記錄 |

---

## 6. 節點類型

| 類型 | 中文 | 行為 |
|------|------|------|
| `NODE_BASIC` | 基礎節點 | 預設已有部分方向可通行 |
| `NODE_SEMI` | 半鎖節點 | **待定**（見下方） |
| `NODE_LOCKED` | 全鎖節點 | 全部出口需要向量才能激活 |
| `NODE_GOAL` | 通關點 G | 抵達即過關 |

### 半鎖節點（待定）

目前有兩個設計方向，尚未選定：

- **方案 A（觸發解鎖）**：玩家走到節點後，自動解鎖新方向或周圍節點
- **方案 B（封閉點回程）**：連通此點的瞬間，系統自動建立回程路線

兩者設計壓力相反：方案 A 考驗「如何抵達」，方案 B 考驗「如何接通」。

## 7. 教學關卡摘要（前三關）

### 第一關：基礎循環

```
S - G
```

直線連通，教玩家完整走過一次「取得 → 移動 → 接通」循環。

### 第二關：觀察即登錄

```
1 x G
|
S - 2
```

- S 向右有死路（2），向上有正解路線（1 → G）
- 玩家需站在 S 登錄「右方向向量」，再移動到接點，接通前往 G 的路線
- **教學核心**：不需要走進死路，站在節點觀察即可登錄

### 第三關：先走後反向取得

```
S - 1
    |
G x 2
```

- 正解：S → 1（取「向左」向量）→ 1 → 2 → 釋放向左 → 接通 G
- **教學核心**：先向前移動，再取得「回頭方向」的向量並異地使用

## 8. 難度結構分析

### 難度來源

| 維度 | 說明 |
|------|------|
| 路徑順序 | 決定何時抵達哪個節點（腳走哪） |
| 方向順序 | 決定在節點登錄什麼、放棄什麼（手畫什麼） |

兩個維度相互依賴，製造解謎深度。

### 庫存 2 格的設計效果

- 依賴鏈無法深度堆疊，必須靠「釋放與取得的交替節奏」製造難度
- 難度天花板：地圖上封閉點（dead-end node）的數量，及其進入順序是否互相干擾
- 兩個封閉點互相干擾，即可達到高難度上限

### 全資訊謎題

- 玩家從一開始就能看到完整盤面
- 挑戰來自「看懂拓撲」和「推導正確順序」之間的距離
- 設計目標：第一眼的直覺解法應為死路，正確解法需要反推

# DIR Level JSON Schema

## 設計原則

1. **位置即方向** — 節點座標決定邊的方向（右=+x、下=+y），不需額外標注
2. **最小化冗餘** — 只記錄設計者決定的資訊，可推導的由引擎計算
3. **邊用節點對表示** — `[A, B]` 中 A 排在 B 前面（字典序），避免重複定義

---

## Schema

```jsonc
{
  // — 關卡後設資料 —
  "id": "1-3",                    // 關卡編號（章-關）
  "name": "先走後反向取得",         // 顯示名稱（可選）
  "inventorySize": 2,             // 庫存上限，預設 2，可省略

  // — 節點 —
  // key = 節點 ID（任意字串，建議短）
  // x, y 採用格子座標（整數），右為 +x，下為 +y
  "nodes": {
    "S": { "x": 0, "y": 0, "type": "start" },
    "1": { "x": 1, "y": 0, "type": "basic" },
    "2": { "x": 1, "y": 1, "type": "basic" },
    "G": { "x": 0, "y": 1, "type": "goal"  }
  },

  // — 邊 —
  // 每條邊是一個物件，nodes 為長度 2 的陣列
  "edges": [
    {
      "nodes": ["S", "1"],        // 連接的兩個節點（字典序排列）
      "active": true              // true = 預設可通行, false = 需要向量接通
    },
    {
      "nodes": ["1", "2"],
      "active": true
    },
    {
      "nodes": ["1", "G"],        // "1" 在 (1,0), "G" 在 (0,1) → 斜向？不對，改用教學關 3
      "active": false
    },
    {
      "nodes": ["2", "G"],
      "active": false,
      "oneway": "2"               // 單向邊：只能從節點 "2" 往 "G" 方向通行
                                   // 值 = 允許出發的那個節點 ID
                                   // 省略 = 雙向
    }
  ]
}
```

---

## 欄位定義

### 頂層

| 欄位 | 型別 | 必填 | 說明 |
|------|------|------|------|
| `id` | `string` | 是 | 關卡唯一識別碼 |
| `name` | `string` | 否 | 關卡顯示名稱 |
| `inventorySize` | `integer` | 否 | 庫存上限，預設 `2` |
| `nodes` | `object` | 是 | 節點字典，key 為節點 ID |
| `edges` | `array` | 是 | 邊的陣列 |

### Node（節點）

| 欄位 | 型別 | 必填 | 說明 |
|------|------|------|------|
| `x` | `integer` | 是 | 格子 x 座標（右為正） |
| `y` | `integer` | 是 | 格子 y 座標（下為正） |
| `type` | `string` | 是 | 見下方節點類型表 |

**節點類型**

| 值 | 對應規格 | 說明 |
|----|----------|------|
| `"start"` | 起點 S | 玩家初始位置，行為同 basic |
| `"basic"` | `NODE_BASIC` | 基礎節點 |
| `"locked"` | `NODE_LOCKED` | 全鎖節點，所有出口需向量接通 |
| `"semi"` | `NODE_SEMI` | 半鎖節點（待定機制） |
| `"goal"` | `NODE_GOAL` | 終點，抵達即過關 |

### Edge（邊）

| 欄位 | 型別 | 必填 | 說明 |
|------|------|------|------|
| `nodes` | `[string, string]` | 是 | 連接的兩個節點 ID |
| `active` | `boolean` | 是 | `true` = 預設可通行；`false` = 需要向量 |
| `oneway` | `string` | 否 | 單向邊的出發節點 ID；省略 = 雙向 |

---

## 方向推導規則

引擎根據兩端節點座標自動推導邊的方向：

```
dx = B.x - A.x
dy = B.y - A.y

if dx > 0, dy == 0 → 水平邊（A 為左端，B 為右端）
if dx == 0, dy > 0 → 垂直邊（A 為上端，B 為下端）
```

- 站在 A 可登錄「向右」或「向下」的向量
- 站在 B 可登錄「向左」或「向上」的向量
- 邊必須是水平或垂直（不支援斜向）

---

## 驗證規則（引擎載入時檢查）

1. 恰好有一個 `type: "start"` 的節點
2. 至少有一個 `type: "goal"` 的節點
3. 每條邊的兩個節點 ID 都存在於 `nodes` 中
4. 每條邊必須是水平或垂直（dx == 0 或 dy == 0，不可同時非零）
5. 不存在重複邊（同一對節點只能有一條邊）
6. `oneway` 的值必須是該邊 `nodes` 中的其中一個

## 教學關卡範例

### 第一關：基礎循環

```
S - G
```

```json
{
  "id": "1-1",
  "name": "基礎循環",
  "nodes": {
    "S": { "x": 0, "y": 0, "type": "start" },
    "G": { "x": 1, "y": 0, "type": "goal" }
  },
  "edges": [
    { "nodes": ["S", "G"], "active": true }
  ]
}
```

### 第二關：觀察即登錄

```
1 x G
|
S - 2
```

```json
{
  "id": "1-2",
  "name": "觀察即登錄",
  "nodes": {
    "S": { "x": 0, "y": 1, "type": "start" },
    "1": { "x": 0, "y": 0, "type": "basic" },
    "2": { "x": 1, "y": 1, "type": "basic" },
    "G": { "x": 1, "y": 0, "type": "goal" }
  },
  "edges": [
    { "nodes": ["S", "1"], "active": true },
    { "nodes": ["S", "2"], "active": true },
    { "nodes": ["1", "G"], "active": false }
  ]
}
```
